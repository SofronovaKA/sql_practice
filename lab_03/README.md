## Задание 1. Общее количество проданных единиц товара

### Текст задания

> Вычислить общее количество проданных единиц товара.

### SQL-запрос

```sql
SELECT 
    COUNT(*) AS total_units_sold
FROM 
    sales;
```

### Ход работы

1. Анализ структуры таблицы sales
* Таблица содержит информацию о каждой продаже.
* Каждая строка соответствует одной проданной единице товара (одной транзакции).

2. Выбор подходящей агрегатной функции
* Для подсчёта количества строк используется функция COUNT(*).
* Альтернатива: COUNT(sales_id) – даст тот же результат, так как sales_id является первичным ключом и не содержит NULL.

3. Составление запроса
* Запрос не требует GROUP BY, так как мы считаем общее количество по всей таблице.
* Добавлен псевдоним total_units_sold для наглядности.

4. Выполнение и проверка
* Запрос возвращает одно число — общее количество продаж.

## Задание 2. Группировка продаж по channel и dealership_id

### Текст задания
> Сгруппировать продажи по channel и dealership_id (сумма продаж).

### SQL-запрос

```sql
SELECT 
    channel,
    dealership_id,
    SUM(sales_amount) AS total_sales_amount
FROM 
    sales
GROUP BY 
    channel,
    dealership_id
ORDER BY 
    channel,
    dealership_id;
```

### Ход работы

1. Анализ требований
* Нужно сгруппировать данные по двум полям: channel и dealership_id.
* Для каждой группы вычислить сумму продаж (sales_amount).

2. Выбор агрегатной функции
* Для суммирования значений используется SUM(sales_amount).

3. Составление запроса
* Указываем в SELECT поля группировки и агрегатную функцию.
* В GROUP BY перечисляем оба поля в том же порядке.
* Добавляем ORDER BY для упорядочивания результата.

4. Учёт особенностей данных
* У онлайн-продаж (channel = 'online') поле dealership_id может быть NULL.
* NULL значения группируются вместе (все онлайн-продажи без дилера попадают в одну группу).

5. Выполнение и проверка
* Результат показывает сумму продаж для каждой пары "канал + дилер".

## Задание 3. Штаты, где клиентов-мужчин больше 100

### Текст задания
> Вывести штаты, где клиентов-мужчин больше 100.

### SQL-запрос

```sql
SELECT 
    state,
    COUNT(*) AS male_customers_count
FROM 
    customers
WHERE 
    gender = 'M'
    AND state IS NOT NULL
GROUP BY 
    state
HAVING 
    COUNT(*) > 100
ORDER BY 
    male_customers_count DESC;
```

### Ход работы

1. Анализ структуры таблицы customers
* Таблица содержит поля: state (штат), gender (пол), а также другие контактные данные.
* Поле gender может содержать значения 'M' (мужской), 'F' (женский) или NULL.

2. Определение последовательности операций
* Шаг 1: Отфильтровать только мужчин (gender = 'M').
* Шаг 2: Исключить записи с неизвестным штатом (state IS NOT NULL).
* Шаг 3: Сгруппировать по штатам (GROUP BY state).
* Шаг 4: Посчитать количество мужчин в каждом штате (COUNT(*)).
* Шаг 5: Оставить только штаты с количеством > 100 (HAVING).
* Шаг 6: Отсортировать по убыванию (ORDER BY ... DESC).

3. Почему HAVING, а не WHERE?
* WHERE применяется до группировки и не может содержать агрегатные функции.
* HAVING применяется после группировки и позволяет фильтровать группы по результату агрегации.

4. Составление запроса
* Фильтры для строк вынесены в WHERE.
* Фильтр для групп вынесен в HAVING.
* Добавлен ORDER BY для сортировки от большего количества к меньшему.

5. Выполнение и проверка
* Результат показывает только те штаты, где количество мужчин превышает 100.
