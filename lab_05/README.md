## Цель работы

Научиться анализировать производительность SQL-запросов, интерпретировать план выполнения (Query Plan) и оптимизировать работу базы данных с помощью индексов (B-tree).

---

## Задачи

1. Изучить принципы работы планировщика запросов в PostgreSQL.
2. Освоить команды `EXPLAIN` и `EXPLAIN ANALYZE` для диагностики производительности.
3. Сравнить последовательное сканирование (`Seq Scan`) и сканирование по индексу (`Index Scan`).
4. Научиться создавать и применять индексы для ускорения выборок.

**Запрос:**
```sql
EXPLAIN ANALYZE
SELECT * FROM sales WHERE sales_amount < 100;
```

### Задание 1.2. Оптимизация запроса локально (создание индекса B-Tree)

**Шаг 1. Создание индекса в локальной базе**
```sql
CREATE INDEX idx_sales_amount ON sales(sales_amount);
```

**Шаг 2. Повторный анализ с индексом**
```sql
EXPLAIN ANALYZE
SELECT * FROM sales WHERE sales_amount < 100;
```

**Скриншот выполнения:**


**Вывод:**
После создания индекса idx_sales_amount тип сканирования изменился с Seq Scan (последовательное сканирование всей таблицы) на Index Scan (чтение по индексу).
Время выполнения сократилось с 1.347 мс до 0.135 мс, то есть примерно в 10 раз.

**Шаг 3. Удаление индекса после проверки**
```sql
DROP INDEX idx_sales_amount;
```

## Вывод

В ходе выполнения лабораторной работы была освоена команда EXPLAIN ANALYZE для анализа плана выполнения запросов. Также было выявлено, что при отсутствии индексов планировщик PostgreSQL использует Seq Scan (последовательное сканирование).
